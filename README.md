# ai-context-kit

Простая папка контекста для Codex и Claude.

Не фреймворк.
Не секретный промпт.
Не замена нормальной документации.

Это маленькая техкарта проекта, которую агент читает перед работой.

Codex и Claude часто начинают новый чат с амнезией.

Они не знают:

- где вход в проект;
- какие файлы главные;
- где лежат секреты;
- какие части лучше не трогать;
- чем проверить результат;
- что уже решили в прошлой сессии.

Из-за этого агент тратит время, перечитывает одно и то же и иногда лезет не туда.

`ai-context-kit` решает это простой папкой внутри проекта.

## Что добавляется в проект

```text
docs/ai_context/
├── PROJECT_MAP.md
├── CURRENT_GOAL.md
├── DANGER_ZONES.md
├── VERIFICATION.md
├── DECISIONS.jsonl
└── SESSION_SUMMARY.md
```

Плюс короткая ссылка из `AGENTS.md` или `CLAUDE.md`.

## Что это дает

Агент быстрее входит в проект.

Он видит карту:

- что это за проект;
- где искать нужный код;
- какие файлы опасные;
- какие проверки запускать;
- что было сделано в прошлый раз.

Это не магия.

Это обычная техкарта.

Как для мастера: где котел, где газ, где электрика, что нельзя трогать без проверки.

## Быстрый старт

Вариант 1. Скриптом:

```bash
./scripts/init-ai-context.sh /path/to/project
```

Вариант 2. Руками:

Скопируй папку `templates/docs/ai_context/` в свой проект.

Потом добавь в `AGENTS.md`:

```md
For faster cold start, read `docs/ai_context/PROJECT_MAP.md` next, then only the relevant `docs/ai_context/*` file for the task.
```

Если у тебя Claude, добавь похожую строку в `CLAUDE.md`.

## Как заполнять

Начни коротко.

Не надо писать книгу.

Хороший `PROJECT_MAP.md` на старте может быть на 40 строк.

Хороший `DANGER_ZONES.md` может быть на 20 строк.

Главное: агент должен быстро понять, куда смотреть и куда не лезть.

## Что нельзя класть в эти файлы

Не клади:

- токены;
- пароли;
- `.env`;
- приватные ключи;
- VPN config;
- реальные IP личных серверов;
- базы данных;
- логи с пользователями;
- приватные чаты.

Пиши так:

```text
Не читать `.env`. Там лежат token и API keys.
```

Не пиши так:

```text
BOT_TOKEN=...
```

## Для каких проектов подходит

Подходит почти для всего:

- Telegram bot;
- web app;
- SaaS;
- WordPress site;
- парсер;
- личный automation repo;
- большой monorepo.

Для маленького проекта хватит одной папки `docs/ai_context/`.

Для большого monorepo можно добавить карты по сервисам.

```text
docs/ai_context/
└── services/
    ├── api.md
    ├── webapp.md
    └── deploy.md
```

## Почему не один огромный промпт

Огромный промпт быстро устаревает.

Файлы рядом с проектом проще поддерживать.

Когда меняется код, меняешь и карту.

Когда новая вкладка стартует, агент читает только нужный кусок.

## Что дальше

Готово в этом наборе:

- скрипт `scripts/init-ai-context.sh`;
- чеклист `NEW_PROJECT_CHECKLIST.md`;
- шаблон `AGENTS.md`;
- шаблоны `docs/ai_context/*`;
- отдельная памятка по секретам в `docs/SECRET_SAFETY.md`;
- минимальный deploy flow в `docs/SAFE_DEPLOY.md`.

Потом можно добавить отдельные примеры для Telegram bot, web app и monorepo.
