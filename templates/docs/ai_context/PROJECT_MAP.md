# PROJECT_MAP

Короткая карта проекта для новой сессии Codex или Claude.

Это роутер, а не архив.

## Что это за проект

Опиши проект в 2-4 строках.

Пример:

```text
Это Telegram bot для личных задач.
Он принимает сообщения, запускает AI-agent и хранит состояние в SQLite.
Проект маленький, но чувствительный, потому что рядом есть токены и runtime DB.
```

## Быстрый вход

1. Прочитай этот файл.
2. Если задача рискованная, прочитай `DANGER_ZONES.md`.
3. Если задача зависит от текущего состояния, прочитай `CURRENT_GOAL.md`.
4. Открой только нужные файлы.
5. Перед финалом смотри `VERIFICATION.md`.

## Карта файлов

| Путь | Роль | Читать когда |
| --- | --- | --- |
| `src/` | Основной код | Меняем поведение |
| `tests/` | Тесты | Проверяем правку |
| `docs/` | Документация | Меняем правила или контракты |
| `.env` | Секреты | Не читать и не печатать |

## Главные потоки

- User action -> handler -> service -> storage -> response.
- Background job -> external API -> database -> notification.

Замени эти строки на реальные потоки проекта.

## Точки поиска

```bash
rg -n "TODO|FIXME|main|handler|route|service" .
rg -n "token|secret|password|api_key|private" . -g '!.env' -g '!*.db'
```

## Правило контекста

Не читать весь проект без причины.

Сначала карта.

Потом зона.

Потом конкретные файлы.

Если этот файл стал большим, вынеси детали в отдельные файлы и оставь здесь только маршруты.
