# DANGER_ZONES

Куда агенту нельзя лезть без причины.

## Секреты

- Не читать `.env`.
- Не печатать tokens, passwords, API keys, private keys.
- Не коммитить DB, logs, sessions, uploads.

## Production

- Не деплоить без отдельной команды.
- Не менять nginx, Docker, ports, cron, systemd как побочный эффект.
- Не удалять данные без подтверждения.

## Данные пользователей

- Не печатать личные сообщения, логи, email, phone, user IDs без причины.
- Если нужно посмотреть структуру, используй schema или fake data.

## Внешние сервисы

- Не отправлять реальные запросы в paid API без причины.
- Не запускать массовые jobs без dry-run или маленького теста.
